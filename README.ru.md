# Apply Tracker

*[English version](README.md)*

Расширение для браузера (Opera, Chrome, Edge, Brave или любой другой Chromium-браузер - поддерживаются Windows, macOS и Linux) для поиска работы на **LinkedIn**, **Indeed** и любом другом сайте с вакансиями:

- Скрывает/показывает карточки вакансий со статусом **Applied** и **Viewed** на LinkedIn (переключатели в попапе, применяются мгновенно).
- Умеет также скрывать вакансии по типу занятости - **Remote**, **Hybrid**, **On-site** (только LinkedIn).
- Добавляет небольшой бейдж **"AA"** (Already Applied) перед названием уже отмеченной вакансии - заметно с первого взгляда, даже если скрытие выключено.
- На Indeed - где нет встроенного отслеживания "applied" - добавляет кнопку **"Mark as Applied"** и такой же бейдж "AA" после отметки.
- На LinkedIn и Indeed рядом с кнопкой Apply/Mark as Applied появляется **выбор CV** - укажите, какое резюме отправляете, до отклика, и оно запишется в колонку "Версия CV" вместе с этим откликом. Список берётся из листа "List CVs" в файле - добавьте туда по строке на каждое резюме, которое хотите видеть в списке. Отключается в попапе ("Record which CV was sent"), если вести учёт по CV не нужно.
- **Любой другой сайт с вакансиями**: плавающая кнопка **«Зафиксировать отклик»** (включается/отключается в попапе) - нажмите её, затем кликните название вакансии и название компании на странице, при желании выберите CV - и отклик записан. Ничего не нужно обучать заранее. Если вы уже откликались на эту вакансию (проверяется по настоящей таблице, а не только по памяти браузера) - кнопка становится зелёной и показывает дату, а рядом с названием вакансии в списке появляется бейдж с той же датой.
- **Скрытие по компании**: небольшая иконка-глаз рядом с вакансией скрывает все объявления от неинтересной компании (переключатель показать/скрыть - в попапе).
- Считает, сколько вакансий отмечено как Applied/Viewed **за текущий день**, и показывает счётчики в попапе.
- Записывает каждый отклик прямо в один Excel-файл (подробности ниже) - без скачиваний и всплывающих окон браузера.
- Если в этой таблице есть старая строка без ссылки (например, вписана вручную) - расширение узнаёт вакансию по названию/компании во время просмотра и само дозаполняет ссылку.

## Быстрый старт

1. Установите [Node.js](https://nodejs.org/) (подойдёт любая текущая версия) - нужен для локального сервера, который пишет Excel-файл.
2. Загрузите расширение: откройте `opera://extensions` (или `chrome://extensions`, `edge://extensions`) → включите **Режим разработчика** → **Загрузить распакованное расширение** → выберите эту папку.
3. Запустите локальный сервер: двойной клик по `server/start-server.bat` (Windows) или `server/start-server.command` (macOS/Linux). Оставьте это окно открытым.
4. Пользуйтесь LinkedIn или Indeed как обычно. Каждый отклик автоматически записывается в **`Applications log.xlsx`**, который создаётся рядом с расширением при первой необходимости - уже с выпадающими списками и листом статистики, настраивать ничего не нужно.

Подробности по каждому шагу - ниже.

## Установка расширения

1. Откройте страницу расширений браузера: `opera://extensions`, `chrome://extensions` или `edge://extensions`.
2. Включите **Режим разработчика** (переключатель, обычно в правом верхнем углу).
3. Нажмите **Загрузить распакованное расширение** и выберите папку, куда вы склонировали/скачали проект.
4. Откройте LinkedIn → Jobs, либо поиск вакансий на Indeed - иконка попапа доступна на обоих сайтах.

Одинаково на Windows, macOS и Linux.

## Локальный сервер (нужен для ведения Excel-журнала)

Расширения браузера не могут напрямую писать файлы на диск, поэтому этим занимается небольшой локальный процесс-компаньон.

1. Установите [Node.js](https://nodejs.org/) (подойдёт любая текущая версия).
2. Запустите сервер:
   - **Windows**: двойной клик по `server/start-server.bat`.
   - **macOS**: двойной клик по `server/start-server.command`. В первый раз macOS может заблокировать запуск ("неизвестный разработчик") - кликните по файлу правой кнопкой → **Открыть** → ещё раз **Открыть** в диалоге.
   - **Linux**: выполните `sh server/start-server.command`, либо `node server/save-server.js`.
   - **Любая ОС, из терминала**: `node server/save-server.js`.

   Должно появиться сообщение, заканчивающееся на `listening on http://127.0.0.1:17845`. Оставьте это окно открытым, пока пользуетесь расширением.
3. Новые отклики дописываются прямо в **`Applications log.xlsx`**, рядом с самим расширением. Если этого файла ещё нет, сервер создаёт его автоматически из встроенного шаблона при первом же отклике - сразу с листом "Отклики" (с выпадающими списками для версии CV и полей Да/Нет), листом "List CVs" (три примерные строки - переименуйте их под свои резюме) и листом "Статистика" на формулах - вручную настраивать ничего не нужно.
4. Расширение заполняет **дату, компанию, вакансию, ссылку, источник и (если вы его выбрали) версию CV** - остальные колонки (ответ, дата ответа, интервью, второй этап, оффер, заметки) вы заполняете сами по ходу общения с работодателем. Расширение никогда не трогает и не перезаписывает эти колонки.

**Автозапуск при входе в систему** (опционально):
- **Windows**: положите ярлык на `start-server.bat` в папку автозагрузки (`Win+R` → `shell:startup`).
- **macOS**: Системные настройки → Основные → Элементы входа → добавьте `start-server.command`.
- **Linux**: добавьте в автозапуск вашего окружения рабочего стола, либо запускайте вручную каждую сессию.

Если сервер не запущен в момент отклика - ничего не теряется, отклик просто дозапишется автоматически, как только сервер снова станет доступен.

### Резервные копии

Каждые 30 минут сервер сохраняет резервную копию `Applications log.xlsx` в `Applications log.backup.xlsx` (в той же папке). Перед этим он сравнивает файлы: если строка есть в резервной копии, но отсутствует в текущем файле (например, что-то его перезаписало или повредило), эта строка автоматически восстанавливается перед обновлением резервной копии. Это одна постоянно обновляемая копия, а не история версий - если нужно откатиться дальше последнего цикла, используйте отдельное средство резервного копирования.

### Открытие таблицы

Кнопка **"Open spreadsheet"** в попапе открывает файл напрямую (либо папку с ним, если это невозможно).

## Совместимость с браузерами

Разработано и проверено на **Chromium-браузерах** - Opera, Chrome, Edge, Brave, Vivaldi и любых других на движке Chromium. Используются стандартные `chrome.*` API расширений и Manifest V3, ничего специфичного для Opera - должно работать одинаково везде.

**Firefox сейчас не поддерживается.** Реализация Manifest V3 в Firefox отличается достаточно сильно (модель фонового скрипта, требование подписи для постоянной установки), чтобы для него потребовались реальные доработки и проверка перед тем, как это можно будет рекомендовать.

## Приватность

Никакие данные никуда не отправляются, кроме локального сервера на вашей же машине (`127.0.0.1`). Никаких внешних серверов, никакой слежки.
